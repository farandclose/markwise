Today you have two ways to give feedback. Both lose.

option one

Retype it in chat. "In section 3, the timeline should be H2 not Q4..." You describe a spot in prose instead of pointing at it, and the location and the intent both blur. The longer the doc, the worse it gets.

option two

Edit the file yourself. The agent gets no structured signal: what changed, what is still open, what needs a response. A raw diff is not a review.

In Google Docs you would just comment on the text. Markwise builds that fluency for the markdown your agent writes.
